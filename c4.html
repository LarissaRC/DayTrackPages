DayTrack - Modelo C4

 O modelo C4 é uma abordagem de abstração para diagramar a arquitetura de software, com base em abstrações que refletem como os arquitetos e desenvolvedores de software pensam e constroem o software. Essa modelagem é dividida em 4 níveis, sendo o nível mais alto o mais próximo da linguagem humana, sendo de fácil entendimento para toda uma equipe. O nível mais baixo está mais próximo do código real, sendo melhor compreendido pela equipe de desenvolvedores.

Nível 1

 Os detalhes não são importantes aqui, pois esta é a visão ampliada do sistema mostrando uma imagem grande da paisagem do sistema. O foco deve estar nas pessoas (atores, funções, personas, etc.) e sistemas de software, em vez de tecnologias, protocolos e outros detalhes de baixo nível. É o tipo de diagrama que pode se mostrar para pessoas não técnicas.

 Vai haver um ator principal, que será o usuário final. Ele terá aesso ao sistema da agenda juntamente com suas funcionalidades. O sistema por sua vez está ligado ao servidor de banco de dados de forma que possa lidar com todos os dados que o ator venha a necessitar.
