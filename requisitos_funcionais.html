DayTrack - Requisitos Funcionais

 Os requisitos funcionais são todos os problemas e necessidades que devem ser atendidos e resolvidos pelo software por meio de funções ou serviços. Tudo o que for relacionado a uma ação a ser feita é considerado uma função. Também é importante lembrar que quanto menos ambíguos e mais objetivos forem os requisitos funcionais, maior será a qualidade do software gerado.

  • Login:
    O sistema permite que o usuário realize login ao fornecer seu e-mail e senha.
  • Cadastro:
    O sistema permite ao usuário se cadastrar, fornecendo os seguintes dados: nome, foto de perfil, email e senha.
  • Visualizar perfil:
    Deve haver um botão que leve o usuário a sua tele de perfil onde poderá visualizar os dados citados no cadastro, com exceção da senha.
  • Editar perfil:
    O usuário poderá atualizar seus dados cadastrados.
  • Lembrar senha:
    O sistema deve enviar um e-mail para o endereço de email do usuário cadastrado que esqueceu sua senha.
  • Deletar perfil:
    O usuário poderá deletar sua conta, sendo imediatamente deslogado e perdendo acesso aos seus eventos cadastrados.
  • Listar eventos:
    O sistema deverá mostrar em uma tela todos os eventos cadastrados pelo usuário em uma lista ordenada conforme os eventos mais próximos de ocorrerem.
  • Pesquisar eventos:
    Ainda na listagem de eventos, é possível pesquisar por um evento específico ao informar parte do seu título.
  • Mostrar o calendário:
    O sistema deve possuir uma tela com um calendário, onde é possível ver todos os dias de um mês selecionado, com todos os eventos cadastrados naquele mês.
  • Trocar mês:
    Na tela de calendário deve haver uma função que permita ao usuário trocar o mês selecionado para que ele possa ver os eventos de outros meses ou cadastrar eventos em meses diferentes.
  • Criar evento:
    O usuário poderá inserir um novo evento, informando o seu título, descrição, data e hora.
  • Visualizar evento:
    O sistema deve mostrar ao usuário os dados do evento que ele cadastrou.
  • Deletar evento:
    O sistema deve deletar os eventos que o usuário desejar, os apagando assim da lista de eventos e do calendário.
  • Notificação:
    O sistema deverá enviar uma notificação ao smartphone do usuário quando um de seus eventos cadastrados for referente ao dia atual.
